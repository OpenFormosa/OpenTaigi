# OpenTaigi 教材體驗設計系統

本設計系統參考
[bergside/awesome-design-skills](https://github.com/bergside/awesome-design-skills)
收錄的 67 套設計技能，抽取所有風格都共用的品質原則，再依台語教材情境選擇
Editorial、Friendly、Lingo、Levels、Paper 與 Spacious 的適用特徵。

不同風格的裝飾語言互相衝突，因此不混用 glassmorphism、brutalism、
neumorphism 等視覺隱喻；所有頁面維持同一套「現代台灣教材」語言。

## 體驗目標

- 教材是主角，導覽與控制不應搶走注意力。
- 看到音節、語詞或語句的同時，就能原位播放真人發音。
- 初學者知道下一步，高程度使用者可快速搜尋、切頁與隱藏提示。
- 桌機保留目錄與設定效率，手機以單欄閱讀和拇指可及操作為優先。

## 基礎規則

- 使用語意 token 管理紙張、文字、靜音文字、主色、成功、警告與錯誤狀態。
- 採 4／8 點間距節奏；教材正文行長以約 70–80 個半形字元為上限。
- 正文使用清楚的無襯線字，教材標題與台文詞頭使用適量襯線字。
- 主要文字與背景需符合 WCAG 2.2 AA；資訊不可只靠顏色表達。
- 互動目標至少 44 × 44 px；支援鍵盤、觸控、指標與可見焦點。
- 優先使用語意 HTML，再補 ARIA；所有圖示按鈕都需有可讀名稱。
- 尊重 `prefers-reduced-motion` 與高對比模式。

## 教材元件

### 閱讀工作階段

- 每頁先顯示一個可完成的學習目標、三個短步驟與本冊進度，不讓使用者只看到一大段教材。
- 正文尾端將「完成本頁」和「下一頁」組成同一個收尾動作，完成狀態保留在裝置中。
- 專注閱讀隱藏冊別側欄、保留教材內容、原位發音、字級與換頁；不另外製造遮住正文的固定工具列。
- 完成進度只反映實際標記的頁數，不使用虛構分數或干擾閱讀的獎勵動畫。

### 關聯與密度

- 標題、頁碼、換頁、閱讀模式、搜尋與字級屬於同一任務，需放在同一控制群。
- 同一控制群內使用 4–8 px 間距；相鄰教材區段使用 12–20 px 間距。
- 語詞與語句採連續清單，共用外框並以分隔線區分，不把每筆資料做成孤立卡片。
- PDF 中的對照表、聲調表與分類表必須重建欄列關係；不可把儲存格依文字擷取順序攤平成段落。
- 聲音必須留在它所解釋的表格列或例字內，播放狀態同時標示整列。
- 留白用於分隔不同學習階段，不用於拆散同一個操作流程。
- 8 冊 217 頁皆需通過頁型巡檢；封面、導讀、發音表、語詞、語句與文章不可共用一個沒有資訊結構的萬用樣板。
- 一般教學頁在 320 CSS px 寬不得讓整頁左右捲動；只有無法重排而不破壞欄列關係的高密度音韻表，可在清楚標示的表格容器內使用內層橫向捲動。
- PDF 的播放記號、字元間雜訊與人工換行屬於來源格式，不可直接混進網頁正文；清理後仍須保留原文、台羅和音檔的對應關係。

### 教材導覽

- 桌機顯示可黏附的冊別目錄與本冊完成度。
- 平板將冊別目錄轉成可橫向捲動的選擇列。
- 手機把四種學習工具放在底部固定導覽，避免與閱讀工具競爭頂部空間。
- 上一頁、下一頁與本冊位置需在教材開頭可見，不必讀到底才換頁。

### 真人發音

- 播放鍵必須依附在對應音節、語詞、例詞或語句上。
- 播放中同時改變按鈕符號、邊框與項目背景，並提供 `aria-pressed`。
- 全域播放器只負責播放進度、速度與關閉，不取代原位控制。
- 未能精準配對到教材區塊的音檔收進頁尾備用清單。

### 提示與答案

- 隱藏提示要清楚說明按下後會顯示什麼，不能只寫「更多」。
- 完整提示模式直接顯示答案並使用非按鈕元素；只有可切換的提示才呈現為按鈕，避免「看似可按但沒有反應」。
- 顯示狀態、焦點狀態與停用狀態都必須明確。
- 長台文、台羅、華語與例詞需可換行，不裁切或依賴 hover。

## 響應式規則

- 980 px 以下：冊別目錄轉為橫向捲動列，教材維持單欄。
- 760 px 以下：工具列改為拇指可及配置，教材卡片改為兩欄核心結構。
- 460 px 以下：減少非必要編號與英文標籤，保留完整教材與操作名稱。
- 固定底部導覽與播放器必須避開 `safe-area-inset-bottom`，且不可互相遮擋。
- 教學模式在 320 CSS px 寬仍維持單欄重排，不要求使用者左右捲動；原版模式則明確放在獨立可捲動區。
- 中文正文以約 40 個全形字為舒適上限，不使用左右對齊，放大字級後仍可自然換行。

這些閱讀規則對照
[W3C Reflow](https://www.w3.org/WAI/WCAG21/Understanding/reflow.html)、
[W3C Visual Presentation](https://www.w3.org/WAI/WCAG22/Understanding/visual-presentation)
和 [W3C Text Spacing](https://www.w3.org/WAI/WCAG22/Understanding/text-spacing)；
可見完成進度則參考
[Khan Academy Course and Unit Mastery](https://support.khanacademy.org/hc/en-us/articles/115002552631-What-are-Course-and-Unit-Mastery-)
把進度和下一個學習動作放在同一條動線。

## 互動狀態

每個控制需視情境具備：

- default
- hover（僅適用支援 hover 的裝置）
- focus-visible
- active／playing
- disabled
- loading
- empty
- error

## 驗收清單

- 所有按鈕可用鍵盤觸發，焦點清楚可見。
- 觸控目標不小於 44 px，手機底部固定元件不遮住內容。
- 文字放大至 135% 時不溢出；長標籤可換行。
- 深色模式與高對比模式仍能辨識播放、完成與停用狀態。
- 關閉動畫偏好時，不執行脈衝或位移動畫。
- 教材原文、音檔、搜尋、提示和完成進度均保留真實資料連結。
